

## DTO 

- Data Transfer Object 
- Padrão de projeto
- Padrão para transferir dados entre subsistemas
- Originalmente conceituado como sendo um objeto objeto simples para forçar uma tipagem de dados
- Nest -> Validação de dados | transformação de dados (diferentemente do original)


> Diferença entre DTO, entity e tipagem

A **Entity** representa o **modelo do banco** e nunca deve ser exposta diretamente ao usuário. O **DTO** define exatamente o que entra e o que sai da aplicação.
